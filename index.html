    Autonomous Remote Control Car

    As a part of my NC State class ECE 306 (Intro to Embedded Systems), I built a remote control line-following car. The car uses a Ti MSP430 micro-controller at its heart, with a secondary control board built to control the power, motors, LCD screen, and IoT functionality. To improve the functionality of the base car, I designed and 3D printed a housing for the sensors to increase the functionality and allow for better positioning. I also wanted to improve the remote control aspect, so I wrote an Android App with a built in Joystick to relay commands to the car over Wi-Fi.

    Chip8mu: Chip-8 Emulator

    Chip8mu is an emulator I wrote in C++ to emulate a Chip-8 interpreter to play games like Pong, Space Invaders, and Cave. It consists of a 64x32 pixel single color screen and 4K of memory. To learn more about the details of the Chip-8 system, see here. Besides core functionality, I also implemented a set of controls for pausing, changing game speed, cycling through color themes, and debugging.

    3D Printed Custom In-Ear Monitors

    In the Summer of 2017, I acquired a Formlabs Form 1+ SLA printer to start making 3D Printed Custom In Ear Monitors (earphones). I built a 3D scanner utilizing photogrammetry technology to scan ear impressions. I then design the shells in 3d modeling software and print the shells on the Form 1+. I use Knowles GV quad balanced armature drivers. The shells are finished with a hypoallergenic lacquer.

    Out-of-Order Execution Superscalar Processor Simulator

    As a part of my NC State class ECE 463 (Advanced Microprocessor Design), I built an OoOE processor simulator in C. The project simulates an N-width instruction life cycle, including the fetch, decode, issue, exectute, writeback, and retire stages. It also includes prefetching and an instruction cache. The project was used to test numerous effects on an instruction pipeline, including the size of the re-order buffer, the size of the issue queue, cache limitation, and prefetching.
